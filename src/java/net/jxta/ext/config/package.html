The JXTA Extension Configuration package provides a number of ready to use, extend and encapsulate configuration utilities.

The principal elements of the Configuration package are:

  • Configurator API
  • Profile Declarative Constructs
  • Configurator UI

The Configurator API is predominately a configuration bean (or property sheet) which entails the spectrum of JXTA configuration attributes. A number of constructors are provided which allows one to instantiate a Configurator object after which any of the available getter/setter methods may be invoked as desired. For most practical use cases, on need only invoke a handful of methods afterwhich the derived PlatformConfig can be created and/or persisted to a specified directory. Prior to generating a PlatformConfig, the Configurator excersises it's current state against normalization, optimization and validation rulesets. The Configurator caller can, optionally, register a number of Optimizer implemenations in order to specialize the derived PlatformConfig per the application requirements. The validation process will generate a ConfigurationException in the event of configuration failures that includes a chained list of exceptions, one for each failure. A prototypical code sample is as follows: 

      URI home = new URI(new File(System.getProperty("user.home", "/.myjxta")));
      Configurator c = new Configurator(home);
      
      c.setSecurity("usr", "pwd");
      
      try {
        c.save();
      } catch (ConfiguratorException ce) }
        ce.printStackTrace();
      }

The Profile constructs allows one to optionally invoke the afore metioned Configurator API via declarative means, as a structured XML file. This model allows one to relatively easily vary the configuration paramaters an applications configuration requirements. A number of prototypical profiles are provided as ready to use templates. A prototypical code sample is as follows: 

      Profile p = new Profile(getClass().getResourceAsStream("/net/jxta/myjxta/resource/profile.xml"));
      URL home = new URI(new File(System.getProperty("user.home", "/.myjxta")));
      Configurator c = new Configurator(p, home);
      
      c.setSecurity("user", "pwd");
      
      try {
        c.save();
      } catch (ConfiguratorException ce) {
        ce.printStackTrace();
      }

The Configurator UI is a optional tier that drives the afore mentioned Profile constructs by providing a series of reusible configuration UI components.
